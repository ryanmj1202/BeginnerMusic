# BeginnerMusic 개발 문맥 정리

작성일: 2026-04-16

## 프로젝트 목표

BeginnerMusic은 한국어 사용자, 특히 작곡 입문자가 웹에서 바로 멜로디를 만들 수 있게 하는 초급자용 작곡 편집기다. 1차 목표는 설명형 대시보드가 아니라 실제 작곡이 가능한 피아노 롤 편집기를 완성하는 것이다.

참고 방향은 Signal MIDI의 편집 화면이다. 구조는 상단 메뉴, 왼쪽 트랙 목록, 중앙 피아노 롤, 하단 세부 편집 패널을 따른다. 단순 복제가 아니라 초급자 친화 흐름과 한국어 UI를 유지한다.

## 사용자 요청과 작업 규칙

- 사용자가 직접 해야 하는 작업, 서버 실행, 외부 계정 연결, 배포, Firebase 연결은 Codex가 실행하지 않고 사용자에게 안내한다.
- 개발 서버 실행은 사용자가 직접 한다. 필요한 경우 `npm.cmd run dev`를 안내한다.
- 확인이 필요한 작업은 사용자가 답하는 방식으로 진행한다. 예: "완료가 되었나요?"
- 파일 저장과 불러오기는 JSON 프로젝트 파일과 MIDI 파일을 모두 지원해야 한다.
- 사용자가 JSON 또는 MID/MIDI 파일을 작업 공간에 드래그해 놓으면 바로 불러와야 한다.
- UI는 어두운 Signal MIDI 스타일을 기준으로 다듬는다.

## 현재 구현된 1차 기능

- React, Vite, TypeScript, Tone.js 기반 편집기 화면
- 어두운 Signal MIDI 유사 레이아웃
- 상단 File 메뉴 안의 새 프로젝트, 열기, 프로젝트 저장
- 왼쪽 트랙 목록, 트랙 추가, 트랙 선택, 악기 선택 팝업
- General MIDI 128개 악기 목록 기반 악기 선택
- 중앙 피아노 롤, 피아노 건반, 마디/박자 그리드
- 선택 트랙 음표와 다른 트랙 회색 고스트 음표 표시
- 클릭으로 음표 생성, 기존 음표 선택, 우클릭 삭제
- 음표 길이 조절 핸들
- 하단 Velocity 패널
- localStorage 자동 저장
- JSON 프로젝트 저장/불러오기
- MIDI 파일 불러오기와 MIDI 템포 적용
- 재생/일시정지/정지 버튼
- Space 키로 일시정지와 이어재생
- 건반과 음표 입력 시 누르고 있는 동안 미리듣기

## 이번 이어 작업에서 반영한 요청

- 빈 피아노 롤 칸을 누른 채 드래그하면 음표가 여러 개 찍히지 않고, 하나의 새 음표가 포인터를 따라 이동한다.
- 기존 음표를 누른 채 드래그해서 위치와 음높이를 바꿀 수 있다.
- 드래그 중 음높이가 바뀌면 미리듣기 소리도 같이 바뀐다.
- 건반을 누르는 동안 색이 바뀌고, 누르고 있는 시간만큼 미리듣기 소리가 유지된다.
- 지우개 도구를 추가했다. 지우개 상태에서 클릭한 채 이동하면 지나간 위치의 음표가 삭제된다.
- 빨간 재생 위치 바가 재생 중에도 requestAnimationFrame으로 계속 움직인다.
- JSON, `.mid`, `.midi` 파일을 편집기 위로 드래그 앤 드롭하면 불러온다.

## 2026-04-17 마무리 수정

- 재생 중 빨간 재생 바가 React 상태를 매 프레임 갱신하지 않도록 바꿨다. 이제 피아노 롤 전체를 계속 다시 그리지 않고 CSS 변수만 직접 갱신한다.
- 음표 드래그 미리듣기에서 음높이가 바뀔 때마다 악기를 새로 만들지 않고, 같은 미리듣기 악기에서 현재 음만 바꾸도록 했다.
- 드래그 중 같은 칸에 머무를 때는 불필요한 프로젝트 상태 업데이트를 하지 않게 했다.
- 이 수정은 재생/미리듣기 중 끊기는 느낌을 줄이기 위한 성능 안정화 작업이다.

## 2026-04-17 추가 조작 개선

- 피아노 롤 상단 타임라인을 클릭하면 빨간 재생 위치가 해당 지점으로 이동한다. 재생 중이면 그 위치부터 다시 이어서 재생한다.
- 피아노 건반을 누른 채 다른 건반으로 드래그하면 현재 건반 음이 계속 미리듣기된다.
- 지우개 삭제는 실제 삭제할 음표가 있을 때만 상태를 갱신하도록 바꿔서 반응 속도를 높였다.
- 멜로디 입력 칸의 커서는 더하기/십자 모양이 아니라 기본 마우스 커서로 바꿨다.
- 트랙을 누르면 악기 선택창이 열리고, 악기 선택창은 카테고리와 악기 목록 2열 구조로 표시된다.

## 2026-04-17 성능 최적화

- 자동 저장은 프로젝트 변경 즉시 `localStorage`에 쓰지 않고 500ms 지연 저장으로 바꿨다. 버튼 클릭, 드래그, 삭제 때마다 큰 JSON 저장이 반복되는 문제를 줄이기 위한 조치다.
- MIDI 재생은 모든 음표를 한 번에 예약하지 않고 짧은 구간씩 lookahead 방식으로 예약한다. 긴 MIDI에서 시작 시 멈칫하거나 소리가 끊기는 문제를 줄이기 위한 구조다.
- 피아노 롤 렌더링에서 음표를 매 줄마다 필터링하지 않고, 음높이별 Map으로 미리 묶어 렌더링한다.
- 재생 중 위치 바 갱신은 React 렌더링 없이 CSS 변수만 갱신하는 방식을 유지한다.

## 2026-04-17 긴 MIDI 발열 대응

- 긴 MIDI에서 `36행 × 전체 스텝 수`만큼 버튼을 렌더링하던 구조를 제거했다. 피아노 롤 격자는 CSS 배경으로 그리고, 클릭/드래그 위치는 좌표 계산으로 처리한다.
- 타임라인도 모든 16분음표 칸을 DOM으로 만들지 않고 마디 단위만 렌더링한다.
- MIDI 재생 lookahead는 매번 전체 음표 배열을 필터링하지 않고, 시작 시간이 정렬된 음표를 `nextIndex`로 앞으로만 훑는다.
- Tone.js PolySynth의 최대 동시 발음 수를 줄여 긴 MIDI에서 CPU 부하를 낮췄다.

## 2026-04-17 건반 드래그 미리듣기 수정

- 건반을 누른 채 다른 건반으로 드래그할 때 이전 건반 소리가 계속 울리지 않도록, 미리듣기 악기는 재생용보다 짧은 release를 쓰게 분리했다.
- 미리듣기 음높이가 바뀔 때 특정 음 하나만 release하지 않고 현재 미리듣기 악기의 열린 음을 모두 release한 뒤 새 음을 재생한다.
- release만으로 이전 소리가 남는 문제가 있어, 건반 드래그로 음높이가 바뀌면 이전 미리듣기 악기를 즉시 dispose하고 새 미리듣기 악기로 현재 음만 재생하게 바꿨다.
- 너무 짧게 스치는 미리듣기 음도 최소 0.25초는 재생되도록 했다. 이전 음 정리는 남은 최소 시간만 기다린 뒤 dispose한다.

## 2026-04-17 피아노 롤 레이어와 템포 UI

- 음표 블록이 피아노 건반 영역 위로 겹쳐 보이지 않도록 건반과 코너 셀의 z-index를 높였다.
- 상단 마디 줄에는 기본 클릭 영역 대신 재생 위치 fill과 handle이 보이는 커스텀 타임라인 seek UI를 추가했다.
- BPM 조절은 하단 transport bar에서 빼고 Velocity 패널 오른쪽 위의 별도 템포 패널로 옮겼다.

## 2026-04-17 트랙 선택과 악기 변경 분리

- 트랙 박스를 누르면 악기 변경 창을 열지 않고 해당 트랙을 편집 대상으로 선택한다.
- 선택된 트랙의 멜로디는 기존 파란색 음표로 보이고, 다른 트랙은 회색 고스트 음표로 유지된다.
- 악기 변경 창은 트랙의 둥근 악기 아이콘 또는 상단 악기 pill을 눌렀을 때 열린다.

## 2026-04-17 트랙 목록 스크롤

- 트랙이 많아질 때 왼쪽 패널이 사이트 아래까지 길어지지 않도록 트랙 목록 영역만 내부 스크롤되게 바꿨다.
- 트랙 툴바와 Add Track 버튼은 고정된 영역으로 유지하고, 가운데 트랙 리스트만 스크롤된다.

## 2026-04-17 트랙 우클릭 메뉴

- 트랙을 우클릭하면 한국어 컨텍스트 메뉴가 뜬다.
- 메뉴 항목은 `트랙 추가`, `트랙 삭제`, `속성`, `트랙 색상 변경`이다.
- 마지막 남은 트랙은 삭제되지 않도록 삭제 버튼을 비활성화한다.
- 트랙 색상은 트랙 왼쪽 라인과 둥근 악기 아이콘 테두리에 반영된다.

## 2026-04-17 우클릭 드래그 삭제

- 피아노 롤에서 우클릭을 누른 채 드래그하면 마우스가 지나간 위치의 음표를 삭제한다.
- 삭제 대상은 현재 선택된 트랙의 음표로 제한한다. 다른 트랙의 회색 고스트 음표는 삭제하지 않는다.
- 음표 블록 위에서 우클릭을 시작해도 삭제 모드가 시작되며, 이어서 드래그하면 닿는 음표들이 삭제된다.

## 2026-04-17 악기 이미지 에셋

- 트랙의 둥근 악기 버튼과 상단 악기 pill에 텍스트 기호 대신 로컬 악기 이미지 에셋을 표시한다.
- 에셋은 `public/instrument-icons`에 SVG 이미지로 저장했다.
- General MIDI 악기군에 따라 피아노, 드럼, 기타, 베이스, 현악, 브라스, 목관, 오르간, 퍼커션, 신스, FX 이미지로 자동 매핑된다.

## 2026-04-17 템포 수정 개선

- 하단 템포 패널의 BPM 입력값이 변경될 때 프로젝트 템포에 바로 반영되도록 했다.
- 자주 쓰는 BPM 프리셋을 select로 선택할 수 있게 추가했다.

## 2026-04-17 재생 바 자동 추적

- 재생 중 빨간 재생 바가 피아노 롤 화면 오른쪽 밖으로 나가려 하면 피아노 롤 가로 스크롤이 자동으로 따라가도록 했다.
- 타임라인을 클릭해 재생 위치를 옮길 때도 해당 위치가 보이도록 스크롤을 맞춘다.

## 2026-04-17 우클릭 삭제 상태 버그 수정

- 우클릭 삭제 후 버튼을 뗐는데도 마우스를 올리면 음표가 삭제되는 문제를 막았다.
- 오른쪽 버튼이 실제로 눌려 있지 않으면 우클릭 삭제 상태를 즉시 해제한다.
- `pointerup`, `pointercancel`, `contextmenu`, `blur`에서도 드래그/삭제 상태를 모두 초기화한다.

## 2026-04-17 추가 렌더링 최적화

- 전역 `keydown`, `pointermove`, `pointerup` 등의 이벤트 리스너가 매 렌더마다 다시 붙지 않도록 핸들러 ref 기반으로 정리했다.
- 선택 트랙 음표, 다른 트랙 음표, 선택 음표, 악기 카테고리/목록 같은 파생 데이터를 `useMemo`로 묶어 불필요한 재계산을 줄였다.
- 제목, 템포, 트랙 선택, 트랙 속성 갱신 시 값이 실제로 바뀌지 않으면 상태를 다시 세팅하지 않게 했다.

## 향후 2차 개발 메모

- 용어 설명: Velocity는 소리 세기, Pitch는 음높이, Volume은 음량처럼 쉽게 설명한다.
- 자동 믹스: 악기 간 음량 균형을 초급자가 이해하기 쉬운 방식으로 자동 조절한다.
- Firebase 저장: 1차 이후 계정 기반 프로젝트 목록 저장으로 확장한다.
- 오디오 출력: 초기에는 가능한 방식으로 다운로드하고, 이후 MP3/WAV 렌더링을 개선한다.
- 악기 소리: 현재 Tone.js 합성음 기반이다. 더 실제 악기에 가까운 소리는 추후 SoundFont 또는 샘플 기반 재생으로 바꾸는 것이 좋다.
- Signal MIDI 세부 UI: Arrange, Tempo, Help, Settings, Sign In 등은 현재 자리만 남겨두고 추후 연결한다.
